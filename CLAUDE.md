# CLAUDE.md — LynxPrompt Action

## Overview
A GitHub Action to sync, validate, generate, and diff AI IDE configuration files (AGENTS.md, CLAUDE.md, .cursor/rules, copilot-instructions.md, etc.) with a LynxPrompt instance.

## Tech Stack
- TypeScript
- Node.js
- Vitest (testing)
- GitHub Actions (composite action)
- @vercel/ncc (bundler)

## Development
```bash
npm install
npm run build        # tsc + ncc bundle -> dist/index.js
npm run test         # vitest run
npm run lint         # type checking
```
The compiled output in `dist/` is committed to the repo (required for GitHub Actions).

Test locally with `act` or by referencing a branch in a workflow:
```yaml
uses: GeiserX/lynxprompt-action@my-branch
```

## Architecture
- `src/main.ts` — entry point, parses inputs, validates token, dispatches to mode handler
- `src/api.ts` — LynxPromptClient wrapping the REST API (`/api/v1/blueprints`)
- `src/modes/{sync,validate,generate,diff}.ts` — each mode is self-contained
- `src/utils/detector.ts` — file detection via glob patterns
- `src/utils/mapper.ts` — blueprint type mapping
- `src/utils/comment.ts` — PR comment posting
- `src/types.ts` — shared interfaces (ActionInputs, ActionMode, Blueprint, etc.)
- `dist/index.js` — bundled with `@vercel/ncc`, MUST be committed
- `action.yml` — GitHub Action definition (inputs: mode, token, api-url, files, visibility, platforms)
- `vitest.config.ts` — test configuration

## Key Rules
- Never hardcode API tokens; they come from action inputs/secrets
- The `dist/` directory must be rebuilt and committed when source changes
- Supports four modes: sync, validate, generate, diff
- API token format is `lp_<64_hex_chars>`
- Default API URL is `https://lynxprompt.com`
- All inputs come from `action.yml` via `@actions/core`
- Config file detection uses glob patterns (default covers AGENTS.md, CLAUDE.md, .cursor/rules/, copilot-instructions.md, .windsurfrules, AIDER.md)
- PR comments use `@actions/github` with a marker comment for idempotent updates

## Known Workarounds
- **`@actions/glob` pinned to 0.5.1** — version 0.6.x ships an ESM-only exports map (`"import"` only, no `"require"`) that `@vercel/ncc` cannot resolve when bundling to CJS. Dependabot is configured to ignore this package. Periodically check if ncc gains ESM exports support or if `@actions/glob` adds a CJS export — then unpin and remove the Dependabot ignore rule.
