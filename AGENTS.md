# LynxPrompt Action

GitHub Action to sync, validate, generate, and diff AI IDE configuration files with LynxPrompt.

## Project Overview

This is a GitHub Action (Node 20, TypeScript) that integrates LynxPrompt with CI/CD pipelines. It operates in four modes: **sync** (upload configs as blueprints), **validate** (check configs are well-formed), **generate** (pull blueprints to repo), and **diff** (compare local vs cloud, post PR comments).

## Architecture

- **Entry point**: `src/main.ts` — parses inputs, validates token, dispatches to mode handler
- **API client**: `src/api.ts` — LynxPromptClient wrapping the REST API (`/api/v1/blueprints`)
- **Modes**: `src/modes/{sync,validate,generate,diff}.ts` — each mode is self-contained
- **Utilities**: `src/utils/` — file detection (`detector.ts`), blueprint type mapping (`mapper.ts`), PR comment posting (`comment.ts`)
- **Types**: `src/types.ts` — shared interfaces (ActionInputs, ActionMode, Blueprint, etc.)
- **Build output**: `dist/index.js` — bundled with `@vercel/ncc`, MUST be committed

## Key Conventions

- All inputs come from `action.yml` via `@actions/core`
- Token format: `lp_` prefix + 64 hex chars
- API URL defaults to `https://lynxprompt.com`, overridable for self-hosted
- Config file detection uses glob patterns (default covers AGENTS.md, CLAUDE.md, .cursor/rules/, copilot-instructions.md, .windsurfrules, AIDER.md)
- PR comments use `@actions/github` with a marker comment for idempotent updates

## Build

```bash
npm install
npm run build  # tsc + ncc bundle → dist/index.js
```

The `dist/` directory is committed to the repo (required for GitHub Actions).

## Known Workarounds

- **`@actions/glob` pinned to 0.5.1** — version 0.6.x ships an ESM-only exports map (`"import"` only, no `"require"`) that `@vercel/ncc` cannot resolve when bundling to CJS. Dependabot is configured to ignore this package. **Periodically check if ncc gains ESM exports support or if `@actions/glob` adds a CJS export — then unpin and remove the Dependabot ignore rule.**
